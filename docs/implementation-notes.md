# Implementation Notes

## 의사결정 기록 (ADR)
치아 연령 회귀 모델의 훈련 스크립트(`src/train.py`)에서 초래된 높은 오차율(7.8년)을 해결하기 위해 다음과 같은 하이퍼파라미터 및 아키텍처 수정을 단행했습니다.

### 1. Batch Size 축소 (64 -> 16)
- **맥락:** 전체 데이터셋이 295장밖에 되지 않아 훈련 데이터가 236장이었습니다. 기존 Batch Size 64에서는 1 Epoch당 최적화 스텝이 4번에 불과했습니다.
- **결정:** 가중치 업데이트 횟수를 늘려 Adam 옵티마이저가 충분한 수렴 시간을 갖도록 Batch Size를 16으로 줄였습니다.

### 2. Epoch 수 증가 (20 -> 150)
- **맥락:** 기존 20 Epoch는 훈련 초기 단계에서 조기 종료된 것과 마찬가지였습니다.
- **결정:** 최대 탐색 범위를 150 Epoch로 대폭 늘려 모델이 훈련 데이터의 피처를 충분히 학습할 수 있도록 수정했습니다.

### 3. Early Stopping (Patience 20) 도입
- **맥락:** 극소형 데이터셋이므로 에포크를 늘릴 경우 필연적으로 모델이 훈련 데이터를 암기하는 과적합(Overfitting) 현상이 발생합니다.
- **결정:** 매 검증 시 Validation MAE를 관찰하고, 20 Epoch 동안 개선이 없으면 강제로 훈련을 중단하는 로직을 도입하여 최상의 가중치 파일이 오버라이트되는 것을 방지했습니다. 실제 53 Epoch에서 조기 종료가 작동했습니다.

### 4. Learning Rate Scheduler 변경 (StepLR -> ReduceLROnPlateau)
- **맥락:** 고정된 주기로 학습률을 절반으로 낮추는 StepLR은 유연한 오차 관리에 적합하지 않았습니다.
- **결정:** Validation Loss가 정체될 때마다 동적으로 학습률을 낮추도록 ReduceLROnPlateau로 교체하여 최소 오차점을 뚫고 들어가도록 최적화했습니다.
